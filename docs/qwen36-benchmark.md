## 1. 테스트 1 — 한국어 단답형

**프롬프트:** *"한국의 수도는 어디이며, 그 역사적 배경을 3문장으로 설명해주세요."*

| 모델 | 엔진 | 생성 속도 | 총 시간 | 한국어 품질 |
|---|---|---|---|---|
| **Qwen3.6-35B FP8** | vLLM | 41.9 tok/s | **1.9 s** | ⚠️ 한자 혼입 ("的战略") |
| bjoernb/gemma4-26b-fast | Ollama | 64.13 tok/s | 17.8 s | ✅ 자연스러움 |
| qwen3:30b | Ollama | 80.33 tok/s | 30.4 s | ✅ 자연스러움 |
| gpt-oss:120b | Ollama | 42.73 tok/s | 36.8 s | ✅ 자연스러움 |

**관찰:**
- 총 시간은 1.9초로 압도적으로 빠름 (모델이 이미 로드된 상태)
- 그러나 **한자 혼입 현상** 발생 ("的战略적 이점") — 한국어 순수성 감점
- tok/s는 ~42로 Ollama qwen3:30b (80 tok/s)보다 느림 (vLLM FP8 vs Ollama Q4 차이)

## 5. 결론

### Qwen3.6-35B-A3B-FP8 (vLLM) 장점
1. **Tool Calling / Agent 워크플로우**: 네이티브 지원으로 에이전틱 코딩에 최적
2. **수능 분석 등 구조화된 태스크**: 5.0/5.0 만점, 기존 모델과 동급 이상
3. **OpenAI 호환 API**: 다양한 클라이언트와 즉시 연동 가능

### 약점
1. **한국어 순수성**: 한자 혼입("的战略") 발생 — thinking 모드나 system prompt로 보완 필요
2. **창작 제약 준수**: thinking 비활성 시 4/9로 낮음 — thinking 활성화 시 개선 예상
3. **tok/s**: Ollama Q4 대비 느림 (42 vs 80). FP8 정밀도 vs Q4 속도 트레이드오프

### 용도별 권장

| 용도 | 권장 모델 |
|---|---|
| **에이전틱 코딩 / Tool Use** | **Qwen3.6 (vLLM)** ⭐ |
| **수능·분석·구조화 태스크** | Qwen3.6 (vLLM) = gemma4 = qwen3:30b |
| **순수 토큰 속도** | qwen3:30b (Ollama, 80 tok/s) |
| **복잡한 창작 제약 준수** | gemma4-26b-fast (Ollama) |
| **한국어 태스크** | gemma4-26b-fast (Ollama) |
| **최고 품질, 속도 무관** | gpt-oss:120b (Ollama) |
